## Core Workflow

1. Identify the niche and product from `product-master.csv`.
2. Reuse existing category/niche decisions before making new ones.
3. If proofreading, run mechanical checks first, then perform a niche-specific editorial pass.
4. If writing sales copy, create both Payhip copy and website copy from the same structured product truth.
5. For single-product ideas, use `single-product-roadmap.csv`.
6. Update the product tracker status after work is complete.

## Store Memory

- Live Payhip store: `https://payhip.com/ContentElevated`
- Products already exist on Payhip as bundles.
- Existing Payhip products currently use older/basic PDFs and will need updated files uploaded after rebrand/proofread.
- Use Payhip for secure checkout and instant digital delivery.
- Use the Content Elevated website as the premium storefront.
- Avoid duplicate Payhip products unless explicitly creating individual/single products.

## Brand Rules

- Brand: Content Elevated.
- Business: premium AI-powered growth bundles for service businesses and creators.
- Output must feel premium, modern, useful, and niche-specific.
- Do not include production labels such as "print-ready," "source content," "review file," or "TODO."
- Do not make fake proof or fake testimonials.
- Secure checkout and instant digital delivery are handled by Payhip.
- Product cover images should be saved locally in `content-elevated-product-os/assets/product-covers/` and used consistently across Payhip, website, and marketing.

## Internal Workspace Rules

- Put Payhip copy, website copy, launch copy, audit notes, upload checklists, URLs, prices, and non-buyer strategy notes in `content-elevated-product-os/internal/[product-slug]/`.
- Keep customer-facing redesigned files in `rebranded-products-sample-direction/[product-slug]/`.
- Do not upload internal documents to Payhip unless they have been rewritten as buyer-facing docs.
- Any file mentioning Gumroad, Etsy, Stan Store, consulting upsells, pricing experiments, or "what still needs to be built" is internal by default.

## Product Copy Must Include

- Product title
- Short hook
- Who it is for
- What is included
- Concrete inventory: PDF/workbook count, customer-facing page count, spreadsheet count, prompt count if confirmed, and template/script/system count where useful
- Business outcome
- Instant download note
- Payhip checkout note
- SEO title and meta description
- Category/tags

## Credit-Saving Rules

- Work in batches by category.
- Research each niche once, then save/use the brief.
- Use scripts for file counts, title checks, repeated labels, and structural QA.
- Do not reread every generated HTML file unless the task requires it.
- Prefer the product tracker and local references over chat history.
