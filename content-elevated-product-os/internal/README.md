# Internal Product Workspace

This folder is for business-side assets that should not be uploaded as customer-facing bundle files unless they are intentionally rewritten for buyers.

## What Lives Here

Each niche should have a folder using the product slug:

`internal/[product-slug]/`

Recommended files:

- `payhip-listing-copy.md`
- `website-product-copy.md`
- `launch-marketing-copy.md`
- `upload-checklist.md`
- `content-audit-notes.md`
- `internal-source-notes.md`
- `payhip-url-and-price.md`

## Customer-Facing Files

Final buyer-facing redesigned documents stay in:

`rebranded-products-sample-direction/[product-slug]/`

Customer-facing folders should not include internal launch strategy, pricing experiments, marketplace notes, website admin notes, or Codex production notes.

## Rule

If a document mentions Payhip upload workflow, listing copy, launch strategy, Gumroad, Etsy, Stan Store, consulting upsells, pricing tests, "what still needs to be built," or production status, it belongs here first.
